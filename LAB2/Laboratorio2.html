Pipeline¶

El pipeline es una herramienta que permite encadenar varias etapas de procesamiento y modelado en un flujo de trabajo coherente. En este caso, se utilizará para combinar el preprocesamiento de datos, la generación de características polinomiales y el modelo de regresión lineal en un solo objeto que se puede ajustar y evaluar de manera eficiente. Asi se aplicaran las mismas transformaciones tanto al conjunto de entrenamiento como al de prueba, evitando fugas de datos y asegurando una evaluación justa del modelo.

def imputar_total_cholesterol_con_ldl(df):
    y = "Total Cholesterol (mg/dL)"
    x = "Estimated LDL (mg/dL)"
    
    k = (df[y] / df[x]).dropna().median()
    df = df.fillna({y: k * df[x]})
    df[y] = df[y].round()
    return df

chol_imputer = FunctionTransformer(imputar_total_cholesterol_con_ldl, validate=False)

def imputar_waist_to_height_ratio_con_circ(df):
    ratio = "Waist-to-Height Ratio"
    circ = "Abdominal Circumference (cm)"
    k = (df[ratio] / df[circ]).dropna().median()
    df = df.fillna({ratio: df[circ] * k})
    df[ratio] = df[ratio].round()
    return df

waist_height_imputer = FunctionTransformer(imputar_waist_to_height_ratio_con_circ, validate=False)

A partir de las curvas de validación y de los valores promedio de RMSE obtenidos con validación cruzada, se observa que el sobreajuste comienza a evidenciarse claramente desde el grado 3 del polinomio. Hasta el grado 2, los errores de entrenamiento y validación se mantienen relativamente cercanos (grado 1: Train ≈ 1.45, Val ≈ 1.35; grado 2: Train ≈ 1.35, Val ≈ 1.53), lo que sugiere un comportamiento aún razonable. Este patrón indica que el modelo empieza a ajustarse excesivamente a los datos de entrenamiento y pierde capacidad de generalización.

  2. Interpretación en términos del trade-off sesgo-varianza. Los resultados pueden interpretarse mediante el trade-off sesgo-varianza: en los grados bajos (1 y 2), el modelo tiene una complejidad moderada, por lo que mantiene un equilibrio razonable entre ajuste y generalización; esto se refleja en errores de entrenamiento y validación relativamente similares, lo que sugiere menor varianza y un sesgo aún aceptable. En cambio, en los grados altos (3 y 4), la complejidad del modelo aumenta significativamente, lo que reduce el error en entrenamiento (menor sesgo aparente) pero incrementa fuertemente el error en validación y su variabilidad entre folds (mayor varianza). En consecuencia, el modelo se vuelve muy sensible a las particularidades de los datos de entrenamiento y aparece el fenómeno de sobreajuste.

  3. Conclusión sobre el control de la complejidad del modelo. En términos de control de complejidad, los resultados sugieren que conviene seleccionar un grado bajo del polinomio (1 o 2) para obtener un mejor desempeño de generalización. En particular, el grado 1 presenta el menor RMSE promedio de validación (≈ 1.3454), por lo que se perfila como la mejor opción entre los grados evaluados. Aunque el grado 2 mejora ligeramente el error de entrenamiento, su error de validación es mayor, por lo que el aumento de complejidad no se traduce en una mejora real del desempeño fuera de muestra.

  4. Observación adicional sobre el grado 4 (RMSE de entrenamiento ≈ 0.0000). Un aspecto importante es que para el grado 4 el RMSE de entrenamiento es prácticamente 0.0000, lo que sugiere que el modelo está ajustando casi perfectamente los datos de entrenamiento. Esto puede ocurrir cuando el modelo es excesivamente flexible frente al tamaño del conjunto de datos o cuando la expansión polinomial genera una representación con demasiada complejidad para la cantidad de observaciones disponibles. Aunque este resultado no implica necesariamente un error en la implementación, sí constituye una señal fuerte de sobreajuste severo, especialmente porque el RMSE de validación sigue siendo alto (≈ 5.95) y con variabilidad considerable. Por tanto, este comportamiento refuerza la necesidad de controlar la complejidad del modelo.

    # Predicciones sobre la muestra
    y_pred_resample = best_poly_lasso.predict(X_resample)
    
    # Calcular RMSE y guardar
    rmse = np.sqrt(mean_squared_error(y_resample, y_pred_resample))
